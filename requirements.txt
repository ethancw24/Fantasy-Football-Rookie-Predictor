# =============================================================================
# requirements.txt
# =============================================================================
#
# PURPOSE:
#   This file lists every external Python library (called a "package" or
#   "dependency") that your project needs to run.
#
# HOW TO INSTALL EVERYTHING AT ONCE:
#   Open your terminal, navigate to your project root folder, and run:
#       pip install -r requirements.txt
#
# WHAT IS pip?
#   pip is Python's package manager — it downloads and installs libraries
#   from the internet (from a website called PyPI) so you can use them
#   in your code.
#
# WHY PIN VERSIONS (the == numbers)?
#   Without version numbers, pip always grabs the newest version of each
#   library.  That sounds good, but newer versions sometimes change how
#   functions work and can break your code silently.  Pinning versions
#   means anyone who installs this project gets the EXACT same libraries
#   you used — so it works the same on every computer.
#
# HOW TO FIND YOUR CURRENTLY INSTALLED VERSIONS:
#   Run this in your terminal:
#       pip freeze
#   It prints every installed package with its version number.
#
# =============================================================================
